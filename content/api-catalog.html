The API-path catalog

Every workflow is an API path you can call: POST /api/run → a standards-traceable report URL. Below are the screens that are live today across three domains, and the domains being onboarded. Nothing here is claimed before it has a firable workflow row.

How the API works

Live paths LIVE

11 live screens across 3 domains, surfacing roughly 100 engineering workflows. Runnable today on the free Open Deck tier — DNV/API-traceable, deterministic, with report URLs published to the public gallery. Each screen takes a short set of clarification inputs, then routes the question to one of the workflows it surfaces (below) and returns a report URL.

Each ref is a screen key in the source-of-truth catalog deckhand/config/deckhand/routing/domain-workflows.yaml. The live screens run on the open-source digitalmodel and worldenergydata engines (tinted chips above are worldenergydata public-data paths). Required inputs are each screen's clarification keys — branch-first inputs like buckle_arrestors, line_material and design_pressure_rating are asked, never guessed. Every call returns a report_url on the public deckhand-sandbox gallery.

A public router path directs a question to the right domain channel, and 4 isolated private-scope delivery paths handle client-confidential work (reports promoted to a client wiki, never the public gallery). Talk to us about private scopes.
