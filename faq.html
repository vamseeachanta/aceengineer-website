What are the typical client engagements?

The typical client engagements are:

  • Existing open source libaries are available for general use: Free
  • New project and no existing library: Time and material basis
  • If customization is required in existing open source libaries: Time and material basis
  • New and cutting edge libraries can be developed with low cost contingent on resource availability.

Algorithm libraries delivered are private property or open source?

Data is completely abstracted from algorithm and the algorithm is released with assumed data. If the algorithm has client IP, it will be delivered to the client via a private repository. Where practical and not constrained by Itellectual Property (IP), algorithms are released as open source libraries for general use.

What are the algorithm best practices?

The algorithms are prepared for the outside world. The best practices followed by all algorithm developers are:

  • Object oriented programming (OOPs) principles.
  • SOLID programming principles. A demo of SOLID programming usinga calculator.
  • Test driven development (TDD) principles.
  • Unit tests and behavior tests.
  • End-to-end behavior tests to practically cover the one-off yet important edge cases.
  • In summary, utilize OOD principles by Robert C. Martin (also known as Uncle Bob).
