Frequently Asked Questions


What are the various project types performed?

The various project types performed to date are:

  • Engineering analysis.
  • Engineering software simulations.
  • Physics-based animations.
  • Data analysis, machine learning, and artificial intelligence.

What are the typical client engagements?

The typical client engagements are:

  • Existing open-source libraries are available for general use: Free
  • New project with no existing library: Time and material basis
  • Customization of an existing open-source library: Time and material basis
  • New and cutting-edge libraries can be developed at a low cost, contingent on resource availability.

Are algorithm libraries delivered as private property or open-source?

Data is completely abstracted from the algorithm, and the algorithm is released with assumed data. If the algorithm contains client intellectual property (IP), it will be delivered via a private repository. Where practical and not constrained by IP, algorithms are released as open-source libraries for general use.

What are the algorithm best practices?

The algorithms are prepared for real-world use. Best practices followed include:

  • Object-Oriented Programming (OOP) principles.
  • SOLID programming principles, with a demo on SOLID design using a calculator.
  • Test-Driven Development (TDD).
  • Unit tests and behavior tests.
  • End-to-end behavior tests to cover critical edge cases.
  • Following OOD principles by Robert C. Martin (Uncle Bob).
